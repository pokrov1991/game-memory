# Новая игра и имя local-игрока

## Имя игрока в local-сборке

В local-сборке профиль игрока хранится в `localStorage` по ключу `orion7:local-player:v1`.

Значение хранится одним объектом:

```json
{
  "id": "local-player-id",
  "name": "Имя игрока"
}
```

`id` создается один раз и используется для записи результата в локальный лидерборд, чтобы у разных игроков не перетиралась одна запись.

`LocalPlatformApi.getUserData()` читает имя и id через платформенное хранилище. Если имя еще не сохранено, local-адаптер возвращает дефолтное имя `Игрок`.

При первом клике на пункты главного меню `Быстрая игра` или `PvP игра` в local-сборке открывается стандартная модалка ввода имени. После сохранения имя записывается в `localStorage`, обновляется пользователь в `UserContext`, затем выполняется изначально выбранный переход.

Перед сохранением имени local-сборка проверяет имя через `PlatformApi.isPlayerNameAvailable()`. Для `LocalPlatformApi` проверка идет в backend лидерборда и не дает сохранить имя, которое уже есть в таблице лидеров у другого `playerId`.

В yandex-сборке эта модалка не используется: имя продолжает приходить через `YandexPlatformApi`.

## Начатая кампания

Начатая кампания определяется функцией `hasStartedCampaign(progress)` из `src/shared/services/platform/defaults.ts`.

Функция сравнивает текущий прогресс с `createDefaultGameProgress()`. Если прогресс отличается от дефолтного состояния, кампания считается начатой.

## Сброс прогресса

Пункт `Новая игра` доступен в главном меню всегда. Если прогресс еще дефолтный, переход во вступление выполняется сразу.

Если прогресс уже отличается от дефолтного, показывается стандартная модалка подтверждения. При нажатии `Начать заново` создается новый `createDefaultGameProgress()`, сохраняется через текущий `platformApi.setGameData()`, записывается в Redux и `UserContext`, затем выполняется переход на `/intro`.

Прямой записи прогресса в `localStorage` из UI нет. Для local и yandex используется один и тот же `PlatformApi`.

## Ручная проверка

- local: первый клик `Быстрая игра` открывает модалку имени, после `Сохранить` выполняется переход.
- local: повторный клик `Быстрая игра` выполняет переход без модалки.
- local: первый клик `PvP игра` открывает модалку имени, после `Сохранить` выполняется переход.
- local: сохраненное имя используется вместо `Игрок`.
- local: `Новая игра` при начатом прогрессе открывает подтверждение и по `Начать заново` сбрасывает прогресс.
- yandex: `Новая игра` при начатом прогрессе открывает подтверждение и сбрасывает прогресс через `YandexPlatformApi`.
- `Отмена` в модалке новой игры закрывает модалку и не сбрасывает прогресс.
