# Ассеты карточек и preload

## Runtime

- Лицевые стороны карточек загружаются из `public/game/card-*.png`.
- Рубашка карточки загружается из `public/game/card.png`.
- `preloadAssets` загружает и декодирует изображения до показа игрового экрана.
- `GameView.loadImages` дополнительно декодирует изображения карточек текущего раунда до отрисовки поля.
- Canvas задается в CSS-пикселях и внутри масштабируется через `devicePixelRatio` до `3`.

## Рекомендуемые размеры

Исходные изображения должны быть минимум в `2x` от самого крупного CSS-размера карточки на целевых мобильных устройствах.

Текущий безопасный минимум:

- Лицевые стороны: `522 x 776` или больше.
- Рубашка: `522 x 776` или больше.
- Для всех лицевых сторон и рубашек нужно сохранять одинаковое соотношение сторон.

Карточки `card-6.png` - `card-10.png` и `card.png` сейчас имеют размер `392 x 582`, поэтому могут выглядеть мягче, если уровень показывает карточки близко к этому размеру на экранах с DPR 2/3.

## Формат

- Для фотореалистичных или рисованных ассетов предпочтителен качественный WebP.
- PNG стоит использовать, когда это нужно для четких краев палитры или прозрачности.
- Не использовать preview-экспорты, которые потом растягиваются canvas.
- Перед заменой ассетов проверять мобильные viewport с DPR 2 и DPR 3.

## Добавление новых ассетов карточек

1. Положить файлы в `public/game`.
2. Добавить их в `public/assets-manifest.json`, чтобы boot preload их декодировал.
3. Сохранить соглашение об именах, которое использует `createCardValues`, или обновить этот helper вместе с манифестом.
4. Запустить `npm run build:local` и `npm run build:yandex`.
