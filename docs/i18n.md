# I18n

Локализация находится в `src/shared/services/i18n`.

## Структура

- `constants.ts` - язык по умолчанию, ключ storage, список поддерживаемых языков и словари.
- `types.ts` - тип `Language`, тип словаря и типизированные ключи переводов.
- `provider.tsx` - `I18nProvider` и hook `useI18n()`.
- `translations/ru.ts` - русский словарь.
- `translations/en.ts` - английский словарь.

Русский язык (`ru`) используется как fallback и язык по умолчанию.

## Использование

В компонентах нужно использовать hook:

```tsx
import { useI18n } from '@/shared/services/i18n'

const { t, language, setLanguage } = useI18n()

return <button>{t('common.save')}</button>
```

`t()` принимает строковый ключ. Если ключ не найден, возвращается сам ключ, чтобы ошибка была заметна в UI.

## Добавление ключа

1. Добавить ключ в `translations/ru.ts`.
2. Добавить тот же ключ в `translations/en.ts`.
3. Использовать ключ через `t('section.key')`.

Структура словарей должна совпадать между языками.

## Добавление языка

1. Создать файл `translations/<language>.ts`.
2. Добавить новый код языка в тип `Language` в `types.ts`.
3. Добавить язык в `SUPPORTED_LANGUAGES` и `translations` в `constants.ts`.
4. Добавить пункт выбора языка на странице настроек.

## Хранение выбранного языка

Ключ хранения: `LANGUAGE_STORAGE_KEY = 'orion7_language'`.

- `local`: язык хранится в `localStorage`.
- `yandex`: язык хранится в `GameProgress.language` через `PlatformApi`.

UI не обращается к `localStorage` или Yandex SDK напрямую. Для чтения и сохранения используется `platformApi.getLanguage()` / `platformApi.setLanguage()`.
